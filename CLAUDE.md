## 1. What this app is

**Router App** is a delivery-courier tool. Couriers scan Temu/Shein/XCargo package labels
with the phone camera; the app extracts the Colombian destination address, geocodes it, and
builds a list of stops. The finished route is saved to a local Room database and exported as
a semicolon-delimited CSV for import into the external **"Routin"** routing app.

**There is NO backend.** Everything runs on-device. The only network call is to Google
Geocoding.

### The pipeline
```
Camera frame (CameraX)
  → ML Kit Latin OCR, cropped to a centered viewfinder band (ScanRegion)
  → ColombianAddressFormatter (on-device regex parse of the label text)
  → Google Geocoding API (pinned to country:CO, region co, language es; LRU-cached)
  → Stop (held in-memory during a session)
  → Room DB (Route + Stops) on save
  → CsvExporter → CSV written to a user-picked folder (Storage Access Framework)
```

### Domain facts worth knowing
- Target market is **Colombia**; addresses use Colombian nomenclature (`Cra.`, `Cl.`, `#`, `Sur`…).
- A **"session"** is the working set of scanned/imported stops held in `CameraViewModel`
  before they are persisted to a `Route`.
- In-session stops use **negative synthetic ids** (`id = -n`); real autogenerated ids are
  assigned only when inserted into Room on save.

./gradlew :app:testDebugUnitTest

- **`GOOGLE_MAPS_API_KEY` must be present in `local.properties`** or the build fails by

- `MainActivity` calls `enableEdgeToEdge()` — see the insets convention in §6.

JVM unit tests live in `app/src/test`. `Stop`/`Route` are plain data classes usable in JVM
tests despite their Room annotations.

## 4. Architecture

Single-Activity (`MainActivity`) → Compose → Navigation-Compose graph (`RouterNavHost`).
Loose **MVVM** with a thin data layer. **No DI framework** — ViewModels new up their own
collaborators with defaulted constructors.

```
ui/
  MainActivity.kt              enableEdgeToEdge(); Theme + Surface hosting RouterNavHost
  navigation/  Routes.kt       route string constants
               RouterNavHost.kt the NavHost graph + ViewModel scoping
  screens/     CameraScreen.kt, RouteHistoryScreen.kt, RouteConfigScreen.kt,
               SaveExportScreen.kt, RouteDetailScreen.kt
  camera/      CameraViewModel.kt   central session state (scan/import)
  detail/      RouteDetailViewModel.kt  one persisted route (view + edit)
  history/     RouteHistoryViewModel.kt list of routes
  theme/       Color.kt, Theme.kt, Type.kt
data/
  ocr/         OcrTextAnalyzer.kt (ImageAnalysis.Analyzer), ScanRegion.kt (crop fractions)
  format/      ColombianAddressFormatter.kt (pure parser), AddressResolver.kt (format→geocode)
  geocoding/   GeocodingApiService, GeocodingResponse (DTOs), GeocodingResult (sealed),
               GeocodingRepository (caching + mapping)
  local/       Route, Stop (entities); RouteDao, StopDao; RouteSummary (projection);
               AppDatabase (singleton + migrations)
  export/      CsvExporter.kt (Stop list → CSV string)
```

### State pattern
- ViewModels expose `private MutableStateFlow` + `public StateFlow`; Composables read via
  `collectAsState()`.
- One-shot UI signals use `MutableSharedFlow` (e.g. `RouteDetailViewModel.editEvents`).
- Per-item in-flight operations are tracked as `Set<Long>` of ids (`deletingStopIds`,
  `editingStopIds`, `deletingRouteIds`) so the UI can disable controls / show progress per row.

## 5. Navigation graph (`ui/navigation/RouterNavHost.kt`)

`startDestination = route_history`. Routes: `route_history`, `route_config`, `camera`,
`camera_edit`, `save_export`, `route_detail`.

| From | To | Notes |
|---|---|---|
| `route_history` | `route_config` ("+ New Route"), `route_detail/{id}` ("Open") | |
| `route_config` | `camera`, `save_export` | Owns a `CameraViewModel` scoped to its NavBackStackEntry |
| `camera` | `save_export` | **Reuses** the `route_config`-scoped `CameraViewModel` |
| `camera_edit/{routeId}` | `save_export` | Adds stops to an EXISTING route; restores that route's city into RouteConfig |
| `save_export` | `route_history` (pop inclusive) | Resolves the `CameraViewModel` from `route_config` or `camera_edit` |
| `route_detail/{routeId}` | `camera_edit/{id}` ("Add More Stops") | Uses its own `RouteDetailViewModel` |

> **CRITICAL:** `CameraViewModel` is shared across `route_config → camera → save_export` by
> passing the **same** `NavBackStackEntry` to `viewModel(entry)`. This is how the in-memory
> session (`sessionStops`, `routeConfig`) survives navigation. Do **not** create a fresh
> `CameraViewModel` per screen.

## 6. Conventions (follow these)

- **State:** `private MutableStateFlow` + `public StateFlow`, read via `collectAsState()`.
  One-shot signals via `MutableSharedFlow`. Per-item progress via `Set<Long>`.
- **Address resolution:** the format→geocode chain MUST go through **`AddressResolver`**.
  Do not re-implement it in a ViewModel.
- **Room access** only through `AppDatabase.getInstance(context)`; DB work on `Dispatchers.IO`.
- **In-session ids:** stops use synthetic negative ids; on persist, `copy(id = 0L, routeId = …, order = …)` before insert.
- **Geocoding** is always Colombia-restricted; `route.city` / `RouteConfig.city` is the
  fallback city when an address carries none.
- **CSV** is semicolon-delimited with a BOM and the fixed Routin header
  (`Name;Address;Latitude;Longitude;Phone;Group;Notes`). Keep `CsvExporter` output
  byte-compatible — its golden test asserts the exact string.
- **Edge-to-edge insets:** every top-level screen applies system-bar insets
  (`Modifier.windowInsetsPadding(WindowInsets.systemBars)` or `Scaffold` contentPadding)
  while keeping its inner `16.dp` padding. **Exception:** `CameraScreen` keeps the camera
  preview full-bleed; only its controls/title get insets (`systemBarsPadding()` on the
  controls Column).
- **File output** uses the Storage Access Framework (`OpenDocumentTree` + `DocumentFile`)
  with a persisted URI permission — never raw file paths.

## 7. Gotchas (these will bite you)

- **ViewModel factory contract.** ViewModels created with the default `viewModel()` factory
  must keep factory-compatible constructors:
  - `AndroidViewModel` subclasses (`RouteDetailViewModel`, `RouteHistoryViewModel`) need a
    single `(Application)` constructor. Adding another param — *even a defaulted one* —
    breaks `AndroidViewModelFactory` **at runtime** (it still compiles). Inject collaborators
    as **internal properties** instead.
  - Plain `ViewModel`s (`CameraViewModel`) work only because **all** constructor params are
    defaulted (Kotlin then emits the no-arg constructor the factory needs). Keep them defaulted.
- **Exactly-one-frame OCR** relies on the `AtomicBoolean` guard in
  `CameraViewModel.shouldAnalyze()`. Preserve it or the analyzer processes every frame.
- **`GOOGLE_MAPS_API_KEY`** missing from `local.properties` = build failure (intentional).
- **DB schema changes** require bumping `@Database(version=…)` and adding a `Migration` in
  `AppDatabase` (currently **v2**; v2 added `Route.city`).
- **Edit safety:** `RouteDetailViewModel.editStop` only overwrites address/coords on a
  *confirmed* geocode; on any failure the stop is left unchanged.

## 8. Key module cheat-sheet

- **`CameraViewModel`** — central session state. Scan flow: `requestScan` → `shouldAnalyze`
  (one frame) → `onOcrResult`/`onOcrFailure`/`onManualAddress` → `AddressResolver.resolve`
  → append `Stop`. Also CSV import (concurrent geocode), session panel (select/remove),
  existing-route mode. Top-level `recentStopsWindow(stops, n=3)` returns the last N stops
  tagged with their real session position (pure, unit-tested).
- **`ColombianAddressFormatter`** — pure parser; the most-tested component. `format(raw,
  fallbackCity)` → `Success(ParsedAddress)` | `NoAddressFound`. `ParsedAddress.displayAddress`
  / `.geocodeQuery`.
- **`AddressResolver`** — shared `format → geocode` chain returning `Ok(name,address,lat,lng)`
  | `Fail(reason)`. Used by both `CameraViewModel` and `RouteDetailViewModel`.
- **`GeocodingRepository`** — LRU cache (cap 50) + status→`GeocodingResult` mapping.
- **`AppDatabase` / DAOs** — `RouteDao` (incl. `getAllWithStopCount(): Flow<RouteSummary>`),
  `StopDao` (insert/update/getByRouteId/delete…).
- **`CsvExporter`** — `exportToCsv` (alias `buildCsv`): BOM + fixed header + `;`-rows,
  lat/lng `%.6f` `Locale.US`, quote-escaping for fields containing `;`.
